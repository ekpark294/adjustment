# 한입정산 작업 내역

## 프로젝트 개요

여러 사람이 함께 주문한 메뉴를 기준으로 참여자별 부담 금액을 계산하는 React 웹입니다.

기본 흐름은 다음과 같습니다.

1. 참여자 이름 등록
2. 메뉴, 가격, 수량 입력
3. 메뉴별로 함께 먹은 사람 선택
4. 메뉴별 분배 금액과 인원별 최종 금액 확인
5. 정산 결과 복사 또는 PNG 이미지 저장

## 기술 구성

- React
- Vite 5.4.21
- `html-to-image`를 이용한 PNG 생성
- 별도 서버나 데이터베이스 없이 브라우저에서 동작

현재 로컬 Node.js 20.11.1과의 호환성을 위해 Vite와 React 플러그인 버전을 고정했습니다.

## 정산 계산 방식

각 메뉴의 1인당 분배 금액은 다음 공식으로 계산합니다.

```text
메뉴별 1인 부담금 = 가격 × 수량 ÷ 해당 메뉴에 선택된 인원 수
```

예시:

```text
가격 25,000원 × 수량 1개 ÷ 8명 = 1인당 3,125원
```

인원별 최종 금액은 해당 사람이 포함된 모든 메뉴의 분배 금액을 합산합니다. 화면에 표시할 때 원 단위로 반올림하므로 전체 합계와 개인별 표시 금액의 합계 사이에 몇 원의 차이가 생길 수 있습니다.

생일자 등을 정산에서 제외하려면 해당 메뉴의 사람 선택 버튼에서 그 사람을 선택 해제합니다. 제외된 사람은 해당 메뉴를 부담하지 않고, 선택된 나머지 인원이 비용을 나눕니다.

개별 수량 모드에서는 참여자별로 `0.5개` 같은 소수 수량을 입력할 수 있습니다. 이때 인원별 최종 금액은 개당 가격에 각 참여자의 개별 수량을 곱해 계산합니다.

## 구현된 기능

### 참여자 입력

- 참여자 이름 추가
- Enter 키로 추가
- 중복 이름 추가 방지
- 참여자 삭제
- 최소 2명 이상일 때 다음 단계 진행
- 참여자를 삭제하면 기존 메뉴의 포함 명단에서도 자동 제거
- 첫 화면에서 임시 저장 내역 불러오기 및 삭제

### 임시 저장

- 브라우저 `localStorage`에 참여자와 메뉴 내역 저장
- 주문 내역 화면에서 `임시 저장` 버튼 제공
- 불러온 내역은 새 항목을 만들지 않고 기존 저장 항목 갱신
- 저장 시각, 참여자, 메뉴 수를 첫 화면에 표시

### 메뉴 입력

- 메뉴명, 개당 가격, 수량 입력
- 메뉴 추가 및 삭제
- 인원별 포함 여부를 버튼으로 선택
- 개별 수량 모드에서 참여자별 소수 수량 입력 지원
- 전체 선택 및 전체 해제
- 새 메뉴는 목록 맨 위에 추가
- 입력 화면 번호는 최신 메뉴부터 역순으로 표시
  - 예: 위에서부터 `3, 2, 1`
- `메뉴 추가` 버튼을 안내 문구 오른쪽에 배치
- `이전`, `정산 결과 보기` 버튼을 긴 메뉴 목록 위로 이동

### 결과 화면

- 총 주문 금액 표시
- 메뉴별 분배 내역 표
- 인원별 최종 정산 금액 표시
- 메뉴 결과표는 입력 순서대로 `1, 2, 3` 출력
- 메뉴별 분배 내역 비고는 참여자 등록 순서대로 표시
- 개별 수량 메뉴는 가격 칸에 총 가격을 표시하고, 분배 금액 칸에는 `개당 ₩O,OOO` 형식으로 표시
- 인원별 정산 금액을 클릭하면 해당 인원이 포함된 메뉴 행을 강조하고 비고의 해당 이름을 굵게 표시
- 인원별 결과를 텍스트로 클립보드에 복사

### 사이트 정보 및 광고 심사 보강

- 첫 화면 하단에 사용 안내, 계산 방식, 개별 수량 설명 콘텐츠 추가
- 푸터에 사용 안내, 개인정보처리방침, 문의 링크 추가
- Google 광고 사용 시 필요한 쿠키 및 제3자 광고 기술 고지를 포함한 정적 개인정보처리방침 페이지 추가
- React 화면으로 접근 가능한 `/guide`, `/privacy` 페이지 추가
- Vercel rewrite 설정으로 `/guide`, `/privacy` 직접 접근 시 React 앱으로 연결
- `sitemap.xml`에 사용 안내와 개인정보처리방침 URL 등록

메뉴별 표 항목:

```text
번호 | 메뉴 | 가격 | 수량 | 인원 | 분배 금액 | 비고
```

비고에는 해당 메뉴에 포함된 참여자 이름을 참여자 등록 순서대로 표시합니다. 개별 수량 모드에서는 이름과 함께 참여자별 수량을 표시합니다.

## PNG 이미지 저장

결과 이미지는 화면 전체를 한 번에 저장하지 않고 다음 세 영역으로 분리했습니다.

1. 총 주문 금액
2. 메뉴별 분배 내역
3. 인원별 정산 금액

파일명 형식:

```text
한입정산-총주문금액-YYYY-MM-DD.png
한입정산-메뉴별분배내역-YYYY-MM-DD.png
한입정산-인원별정산금액-YYYY-MM-DD.png
```

메뉴 이미지는 화면의 제목, 메뉴 개수, 스크롤바, 저장 버튼을 제외하고 `<table>` 자체만 저장합니다. 인원별 이미지는 제목과 저장 버튼을 제외하고 결과 카드 자체만 저장합니다. 총액 이미지는 카드 배경과 문구가 잘리지 않도록 별도 높이와 여백을 적용했습니다.

## 모바일 대응

- 600px 이하 모바일 레이아웃 적용
- 380px 이하 초소형 화면 추가 대응
- 입력 필드를 1~2열로 재배치
- 모바일 입력 시 브라우저 자동 확대 방지
- 긴 이름과 금액 겹침 방지
- 카드와 입력 영역의 가로 넘침 방지
- 메뉴 결과표 가로 스크롤 지원
- 터치 버튼 높이 확대
- 결과 버튼 모바일 재배치

## 실행 방법

프로젝트 폴더에서 다음 명령을 실행합니다.

```powershell
npm install
npm run dev
```

기본 개발 서버 주소:

```text
http://localhost:5173
```

프로덕션 빌드 확인:

```powershell
npm run build
```

현재까지 모든 변경 후 Vite 프로덕션 빌드를 통과했습니다.

## Vercel 배포 준비

현재 프로젝트 폴더는 Git 저장소가 아니며 Vercel 로그인 정보도 없는 상태입니다. Vercel CLI 최신 버전은 로컬 Node.js 20.11.1보다 높은 버전을 요구하므로 호환 CLI를 사용합니다.

로그인:

```powershell
npx vercel@37.14.0 login
```

로그인 후 미리보기 배포:

```powershell
npx vercel@37.14.0
```

프로덕션 배포:

```powershell
npx vercel@37.14.0 --prod
```

장기적으로는 Node.js를 최신 LTS 버전으로 올리고 최신 Vercel CLI를 사용하는 것이 적절합니다.

## 주요 파일

- `src/App.jsx`: 화면 단계와 공통 상태 연결
- `src/components/`: 헤더, 푸터, 참여자·주문·결과 단계 화면
- `src/components/InfoContent.jsx`: 첫 화면 하단 안내 콘텐츠
- `src/hooks/useDrafts.js`: 로컬 임시 저장 관리
- `src/utils/settlement.js`: 정산 금액 계산과 금액 표시
- `src/utils/imageDownload.js`: PNG 이미지 생성 및 다운로드
- `src/styles.css`: 전체 UI와 반응형 스타일
- `src/main.jsx`: React 진입점
- `vite.config.js`: Vite React 설정
- `package.json`: 실행 명령과 패키지 구성

이미지 저장 버튼은 생성 단계의 추정 진행률을 표시하며, 파일 생성이 완료되면 100%로 표시합니다. `html-to-image`가 내부 렌더링 진행률을 제공하지 않으므로 중간 퍼센트는 실제 처리량이 아닌 대기 상태 안내용입니다.
